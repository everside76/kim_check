# 김치옥 MOT 점검 웹앱 — 사용 설명서

매장에서 스마트폰으로 MOT 서비스 체크리스트를 점검하고, 결과를 **구글 스프레드시트로 자동 전송**하는 모바일 웹앱입니다.
별도 앱 설치 없이 **인터넷 주소(URL) 하나**로 동작합니다.

🥬 **김치옥 전용** 앱입니다. (고반식당은 별도 앱으로 운영)

원본: `김치옥_MOT_체크리스트_v2.xlsx` (MOT 7단계 · 35문항)

## 🚀 빠른 시작 (3단계)

1. **구글 시트 연동 설정** (아래 1번 — 한 번만 하면 됨)
2. **GitHub Pages로 앱 배포** (아래 2번 — 한 번만 켜면 됨)
3. **점검 → 결과 전송** (아래 3번)

> **기본 구글 시트 연동 URL이 앱에 포함되어 있어** 별도 설정 없이 결과가 전송됩니다.
> 아래 1번은 *다른 구글 시트*로 보내고 싶을 때만 참고하세요.

## 1️⃣ 구글 스프레드시트 연동 (선택 — 다른 시트로 보낼 때만)

> 기본 연동 URL이 이미 들어 있습니다. 아래는 **본인 소유의 다른 구글 시트**로 결과를 모으고 싶을 때만 따라 하면 됩니다.

### ① 구글 시트 새로 만들기
- [sheets.google.com](https://sheets.google.com) 접속 → **빈 스프레드시트** 생성
- 시트 이름을 `김치옥 MOT 점검결과` 등으로 지정 (자유)

### ② Apps Script 열기
- 상단 메뉴 **[확장 프로그램] → [Apps Script]** 클릭

### ③ 코드 붙여넣기
- 편집기에 있던 `function myFunction() {}` 등 **기존 내용을 전부 삭제**
- `google-apps-script.gs` 파일을 메모장으로 열어 **전체 복사 → 붙여넣기**
- 상단 💾 **저장** (또는 Ctrl+S)

### ④ 웹 앱으로 배포
- 우측 상단 파란 **[배포] → [새 배포]** 클릭
- 톱니바퀴 ⚙️ → **유형 선택: "웹 앱"**
- 설정값:
  - **설명**: `MOT 수신` (자유)
  - **실행 계정(다음 사용자로 실행)**: **나**
  - **액세스 권한(액세스 권한이 있는 사용자)**: **모든 사용자**
- **[배포]** 클릭
- 처음이면 **권한 승인** 창이 뜸 → 본인 구글 계정 선택
  - "Google에서 확인하지 않은 앱" 경고가 나오면
    → **[고급] → [(프로젝트 이름)(으)로 이동(안전하지 않음)]** → **[허용]**
    *(내가 만든 내 시트로 보내는 것이라 안전합니다.)*

### ⑤ 웹 앱 URL 복사 → 앱에 입력
- 배포 완료 화면에 나오는 **웹 앱 URL** 복사
  - `https://script.google.com/macros/s/........./exec` 형태
- 점검 앱 첫 화면 → **설정** → **Apps Script 배포 URL** 칸에 붙여넣고 **저장**

> 💡 **확인 팁**: 복사한 URL을 브라우저 주소창에 붙여넣어 열었을 때
> `"김치옥 MOT 수신 서버가 정상 동작 중입니다."` 가 보이면 정상입니다.

## 2️⃣ GitHub Pages로 앱 배포하기 (무료 호스팅)

이 프로젝트는 GitHub 저장소 **[everside76/kim_check](https://github.com/everside76/kim_check)** 에 올라가 있습니다.
GitHub Pages를 한 번만 켜면, 직원 누구나 아래 주소로 점검 앱에 접속할 수 있습니다.

👉 **앱 주소: `https://everside76.github.io/kim_check/`**

### 최초 1회 — Pages 켜기
1. 저장소 페이지 → 상단 **Settings** 클릭
2. 왼쪽 메뉴 **Pages** 클릭
3. **Build and deployment → Source**: `Deploy from a branch` 선택
4. **Branch**: `main` / 폴더 `/ (root)` 선택 → **Save**
5. 1~2분 뒤 위 주소(`https://everside76.github.io/kim_check/`)로 접속되면 완료

### 앱을 수정한 뒤 반영하기
- `index.html` 등을 고치고 **커밋 & 푸시**하면, GitHub Pages가 자동으로 다시 배포합니다.
- 직원들은 **새로고침**만 하면 최신 버전이 적용됩니다. (URL은 그대로)

```
git add -A
git commit -m "수정 내용"
git push
```

### 📱 홈 화면에 추가 (앱처럼 사용)
직원들이 위 주소에 접속한 뒤:
- **아이폰(Safari)**: 공유 버튼 → "홈 화면에 추가"
- **안드로이드(Chrome)**: 우측 ⋮ 메뉴 → "홈 화면에 추가"
- 추가하면 아이콘(김치옥 MOT)을 눌러 일반 앱처럼 실행됩니다.

## 3️⃣ 점검하고 결과 전송하기

### ① 점검 진행
1. 첫 화면에서 **지점명 · 점검자 · 주문 방식**(직접주문/테이블오더) 입력
   - 주문 방식에 따라 전용 점검 항목이 자동으로 바뀝니다.
   - 점검 구분(본사점검/SV점검), 피점검자(대상 직원)도 함께 기록할 수 있습니다.
2. **점검 시작하기** → 단계별 항목을 **○ 이행 / △ 미흡 / × 불이행** 으로 체크
   - 각 항목 **📝 비고**에 특이사항 기록 가능
   - 상단 단계 칩(입점·주문…)을 누르면 해당 단계로 빠르게 이동
3. 하단 **절대 금지 10계명** 에서 발생 항목 체크 (1건당 −5점)
4. 하단 **결과 보기 →** 클릭

### ② 결과 확인 & 전송
- 총점/등급, 단계별 감점, 감점 항목 상세가 자동 계산됩니다.
- **종합 코멘트** 작성 후 **구글 시트로 결과 전송**
- 전송되면 구글 시트의 `점검요약` · `항목상세` 탭에 자동으로 행이 추가됩니다.
- 인터넷이 안 되면 결과는 휴대폰에 저장되고, **📋 점검 이력**에서 나중에 **🔁 재전송** 가능.
- 필요하면 **⬇️ 엑셀(CSV) 저장**으로 파일을 따로 받을 수도 있습니다.

## 📊 구글 시트에 쌓이는 데이터

### `점검요약` 탭 (점검 1건 = 1행)
전송일시 · 점검ID · **브랜드** · 지점명 · 피점검자 · 점검자 · 점검일 · 요일 · 점검구분 · 주문방식 ·
**총점 · 등급 · 가산점** · 이행수 · 미흡수 · 불이행수 · 미응답수 · 10계명위반수 · 총항목수 · 종합코멘트

### `항목상세` 탭 (항목 1건 = 1행)
전송일시 · 점검ID · **브랜드** · 지점명 · 점검일 · No · 단계 · 구분 · 항목 · 응답 · 감점 · 가산 · 비고
*(10계명 위반 내역도 함께 기록됩니다.)*

> 이 두 탭을 이용해 매장(지점)별·기간별 평균 점수, 자주 미흡한 항목 등을 피벗테이블로 분석할 수 있습니다.
> (`브랜드` 컬럼은 항상 "김치옥"으로 기록됩니다.)

## 🧮 점수 계산 기준

- **항목 평가**: ○ 이행 = 감점 0 / △ 미흡 = **배점의 절반** / × 불이행 = **배점**
- **문항별 가중 배점**: 항목 중요도에 따라 불이행 시 −2 또는 −4 (미흡은 그 절반)
- **100점 만점 환산**: 적용 항목 전체 배점을 100점으로 보고, 감점 비율만큼 차감
  - 전부 이행 = 100점 / 전부 불이행 = 0점
- **10계명 위반**: 1건당 **−5점** (환산 점수에서 추가 차감)
- **등급**: S(90%↑) / A(80~89%) / B(70~79%) / C(70% 미만)
- 미응답 항목은 감점에 포함되지 않습니다(결과 화면에 건수 표시).

## ❓ 자주 묻는 질문 / 문제 해결

**Q. 전송이 계속 실패해요.**
- 설정의 URL이 `.../exec` 로 끝나는지 확인 (`/dev` 아님)
- Apps Script 배포 시 액세스 권한을 **"모든 사용자"**로 했는지 확인
- 코드 수정 후에는 **[배포] → [배포 관리] → 편집(연필) → 새 버전 → 배포** 로 다시 배포해야 반영됩니다.

**Q. '구글 시트 URL을 확인해 주세요' 라고 나와요.**
- 기본 연동 URL이 포함되어 있어 보통은 나오지 않습니다. 설정에서 URL을 지웠거나 잘못 입력한 경우이니, **설정 → Apps Script 배포 URL** 칸을 비워 저장(기본 URL 사용)하거나 올바른 `.../exec` 주소를 입력하세요.

**Q. 결과가 휴대폰에만 있고 시트에 없어요.**
- 전송 실패로 로컬에 저장된 상태입니다. **📋 점검 이력 → 🔁 재전송** 하세요.

**Q. 점검 이력이 사라졌어요.**
- 이력은 **그 휴대폰의 브라우저**에만 저장됩니다. 브라우저 캐시/데이터를 지우면 함께 삭제됩니다.
  영구 보관은 구글 시트 전송 또는 CSV 저장을 이용하세요.

**Q. 항목을 수정·추가하고 싶어요.**
- `index.html`의 `const ITEMS_KIMCHIOK = [...]` 부분을 편집하면 됩니다. (확실치 않으면 요청 주세요.)
